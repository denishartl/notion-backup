## Project Notes

**notion-backup: Fixed database/data_source API distinction**

Critical issue: Notion API has two different concepts:- `database_id` → `/v1/databases/{id}` endpoints- `data_source_id` → `/v1/data_sources/{id}` endpoints

When pages have `parent.type == "data_source_id"`, they're rows in a data source, NOT a database. Must use:
- `client.data_sources.retrieve()` instead of `client.databases.retrieve()`
- `client.data_sources.query()` instead of `client.databases.query()`

Also fixed null property handling in markdown converter - some Notion pages have `None` values for properties which caused "'NoneType' object has no attribute 'get'" errors.
